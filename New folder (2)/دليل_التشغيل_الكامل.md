# 🏥 دليل تشغيل بوابة مستندات الأطباء
## مستشفيات الحمادي — Doctor Documents Portal

---

## 📁 الملفات المطلوبة

| الملف | الوصف |
|-------|-------|
| `index.html` | الواجهة الرئيسية للبورتال |
| `style.css` | ملف التنسيقات |
| `script.js` | منطق البرنامج وقاعدة بيانات الأطباء |
| `Code.gs` | كود الخادم (Google Apps Script) |

---

## الخطوة 1 — إنشاء Google Drive Folder

1. افتحي [drive.google.com](https://drive.google.com)
2. أنشئي فولدر جديد اسمه: **`Doctors Documents Uploads`**
3. افتحيه ثم انسخي الـ ID من الرابط:
   ```
   https://drive.google.com/drive/folders/►FOLDER_ID◄
   ```
4. احتفظي بالـ ID

---

## الخطوة 2 — إنشاء Google Sheet

1. افتحي [sheets.google.com](https://sheets.google.com)
2. أنشئي شيت جديد اسمه: **`Al Hammadi Doctors Portal`**
3. انسخي الـ ID من الرابط:
   ```
   https://docs.google.com/spreadsheets/d/►SHEET_ID◄/edit
   ```
4. احتفظي بالـ ID

---

## الخطوة 3 — إعداد Apps Script

1. افتحي [script.google.com](https://script.google.com)
2. اضغطي **"مشروع جديد"**
3. سمّي المشروع: `Doctor Portal - AlHammadi`
4. احذفي الكود الموجود بالكامل
5. انسخي محتوى ملف `Code.gs` كاملاً والصقيه
6. عدّلي السطرين التاليين في أعلى الكود:
   ```javascript
   const SHEET_ID  = 'ضعي SHEET_ID هنا';
   const FOLDER_ID = 'ضعي FOLDER_ID هنا';
   ```
7. اضغطي **💾 حفظ** (Ctrl+S)

---

## الخطوة 4 — إعداد Script Properties (كود الإدارة)

1. من القائمة العلوية: **`تشغيل`** ← **`setupProperties`**
2. ستطلب صلاحيات — اقبليها:
   - **"مراجعة الأذونات"** ← اختاري حسابك
   - **"متقدم"** ← **"الانتقال إلى Doctor Portal"**
   - **"سماح"**
3. بعد التشغيل: كود الإدارة محفوظ بأمان
4. **لتغيير الكود:** عدّلي هذا السطر في `Code.gs` قبل التشغيل:
   ```javascript
   const ADMIN_CODE = 'AlHammadi@Admin2025';  // ← عدّلي هذا
   ```

---

## الخطوة 5 — إنشاء الشيتات

1. من القائمة: **`تشغيل`** ← **`setupAllSheets`**
2. سيُنشئ 4 شيتات تلقائياً:
   - **Doctors** — بيانات الأطباء وحالة الاكتمال
   - **Uploads** — سجل الملفات المرفوعة
   - **Logs** — سجل الأنشطة
   - **Dashboard_Summary** — ملخص الإحصائيات

---

## الخطوة 6 — نشر كـ Web App

1. من الأعلى: **"نشر"** ← **"نشر جديد"**
2. اضبطي:
   - **النوع:** تطبيق ويب
   - **الوصف:** Doctor Portal v1
   - **تشغيل بصفتي:** أنا (your email)
   - **من يمكنه الوصول:** **الجميع** ✅ (مهم جداً)
3. اضغطي **"نشر"**
4. **انسخي الرابط** — يبدأ بـ:
   ```
   https://script.google.com/macros/s/XXXX/exec
   ```

---

## الخطوة 7 — ربط البورتال بالرابط

1. افتحي `script.js`
2. ابحثي عن:
   ```javascript
   const GAS_URL = 'YOUR_APPS_SCRIPT_URL_HERE';
   ```
3. استبدليها برابط الـ Web App:
   ```javascript
   const GAS_URL = 'https://script.google.com/macros/s/XXXX/exec';
   ```
4. احفظي الملف

---

## الخطوة 8 — رفع البورتال ونشره

### الخيار أ: GitHub Pages (مجاني — موصى به)
1. أنشئي repo جديد في [github.com](https://github.com)
2. ارفعي الملفات الثلاثة: `index.html` + `style.css` + `script.js`
3. من الإعدادات ← **Pages** ← فعّلي من `main` branch
4. الرابط: `https://username.github.io/repo-name`

### الخيار ب: Google Sites
1. افتحي [sites.google.com](https://sites.google.com)
2. أنشئي موقعاً جديداً ← صفحة جديدة ← **Embed** ← HTML
3. انسخي كامل محتوى `index.html`

### الخيار ج: أي استضافة ويب
- ارفعي الملفات الثلاثة على أي استضافة تدعم HTML

## 🛠️ مشاكل شائعة

| المشكلة | الحل |
|---------|------|
| "خطأ في الإذونات" | تأكدي من اختيار "الجميع" في خطوة النشر |
| الملف لا يُرفع | تأكدي من FOLDER_ID |
| البيانات لا تظهر في Sheet | تأكدي من SHEET_ID |
| "Unknown action" | تأكدي من نسخ Code.gs بالكامل |
| لوحة الإدارة فارغة | شغّلي `setupAllSheets` أولاً |
| بعد أي تعديل على Code.gs | أعيدي النشر: "نشر" ← "إدارة النشرات" ← "تحرير" ← "نشر" |
